# 味噌汁市ナレッジベース INDEX

このリポジトリは、Minecraft架空都市「味噌汁市」に関する情報の唯一の情報源です。
台本・原稿・記事を生成するAIは、作業前に必ずこのファイルを読んでください。

## 二つの世界（最重要ルール）

このKBは情報を二つの世界に分けて管理します。

- `lore/` — **作中世界の事実**。味噌汁市が実在する都市であるかのように書かれた設定（地理・作中史・行政・名所）。
- `project/` — **現実の事実**。味噌汁市を制作しているコミュニティ・経緯・運営・実績。

生成AIは出力前に、どの「語りのモード」で書くかを必ず決めてください。

| モード | 用途の例 | 使ってよい事実 |
|---|---|---|
| インユニバース | 観光ガイド、市の公式文書風、シティガイド誌面 | lore/ のみ。project/ の情報を作中に持ち込まない |
| メタ | 建築解説・メイキング動画、コミュニティ紹介、インタビュー | project/ が主。lore/ に触れるときは「設定では〜」と明示する |
| 混在 | 同人誌・特集など複数記事で構成されるもの | 記事単位でどちらかのモードに振り、1記事の中で混ぜない |

禁止事項:

- 現実の出来事（メンバー・サーバー・掲載実績など）を作中史として書かない。
- 作中設定（架空の年代・人口など）を現実の説明として書かない。
- 作中の年代と現実の年月を、区別なく同じ文の中で混ぜない。

## AIへの利用ルール

1. このKBに書かれていない事実を創作しない。
2. `【要記入：…】` は「未確定情報」として扱い、推測で補完しない。出力に必要な場合は「（要確認）」と明示する。
3. 固有名詞の読み方は `glossary.md` に従う。glossaryにない読みを勝手に決めない。
4. 台本の出力形式は `voice/formats/` 配下の定義に従う。
5. `project/production-notes/` の status が「構想区域（凍結）」の地区について、現実の開発が進行中であるかのような表現をしない。
6. `templates/` は人間用の複製元であり、生成の参照元として読まない。

## ディレクトリマップ

- `glossary.md` — 用語と読み仮名（両世界共通。誤読防止）
- `lore/` — 作中世界の正典
  - `city.md` — 市の基本設定（架空の地理・規模）
  - `history.md` — 市史（作中の歴史のみ）
  - `districts/` — 地区の設定（1地区1ファイル）
  - `landmarks/` — 主要建築の設定（1建物1ファイル）
- `project/` — 現実の正典
  - `overview.md` — コミュニティ・体制・制作環境
  - `history.md` — プロジェクト史と界隈史における位置づけ
  - `production-notes/` — 制作ノート（開発経緯・ネタ・撮影メモ。対象1つにつき1ファイル）
- `voice/` — 発信の声
  - `style-guide.md` — 口調・モード別文体・NG集
  - `formats/` — 動画フォーマット定義（台本の型と出力スキーマ）
- `examples/` — 成績メモ付きの過去台本（few-shotのお手本）
- `log/` — 月別の開発ログとネタの記録
- `templates/` — 新規ファイルの複製元（人間用）

## タスク別・読むべきファイル

| タスク | モード | 読む順番 |
|---|---|---|
| シティガイド・市の公式文書風の原稿 | インユニバース | INDEX → voice/style-guide.md → lore/ 配下のみ |
| Shorts台本（建築解説・メイキング） | メタ | INDEX → voice/style-guide.md → voice/formats/shorts-sandwich.md → project/production-notes/対象 → 設定に触れる場合は lore/districts/対象 → examples/ から類似1〜2本 |
| 長尺ツアー台本 | メタ（指定があれば従う） | 上記すべて ＋ voice/formats/city-tour.md ＋ 必要に応じて lore/history.md・project/history.md |
| 設定資料集動画（作中世界の紹介） | インユニバース | INDEX → voice/style-guide.md → voice/formats/lore-showcase.md → lore/ の対象ファイル → examples/ から類似1〜2本 |
| 同人誌・特集記事 | 混在 | 記事ごとにモードを決め、上の該当行に従う |
| 設定の確認・Q&A | — | 作中のことは lore/ のみ、現実のことは project/ のみを根拠にする |

## 更新ルール（人間向け）

- 新しい事実・ネタは log/ に記録し、編集会議で lore/ または project/ へ昇格させる。**昇格先の世界をその都度確認する。**
- ファイルを更新したら、フロントマターの updated を更新日に変更する。
- 地区の status（開発区域／保存地区／構想区域（凍結））は都市計画マスタープランの決定と連動させ、project/production-notes 側でのみ管理する。
